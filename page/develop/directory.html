# 2、目录说明

# 2.1 开发

和开发相关的目录主要是 src 目录，在该目录下是整个 SDK 的源码。为了针对不同框架和平台开发不同的采集功能、src 目录根据运行平台的不同设计了不同的目录用于开发。

core 目录是整个 SDK 的核心部分，用于编写在所有平台都会使用到的核心通用功能，对于只在某一个平台会用到的特异性功能代码将编写到 platforms 目录下的对应平台目录中。

platforms 目录下的文件夹根据 SDK 使用平台和框架的不同进行了区分、每个目录下的文件只编写 SDK 运行在该平台下的代码，目前只有 nativejs 目录和 vue 目录，后续如需拓展到 React、Angular等其他框架或平台时只需在 platforms 目录下新建对应的文件夹并按照约定开发即可。

platforms 目录下的所有源码文件全都继承自 core 目录下的 class，core 目录下的基类定义全平台通用功能，platforms 目录下的子类继承 core 目录下的基类然后扩展实现特定平台的功能。

综上新增功能时应当按照如下顺序开发：

  1. 在 core 目录下创建基类、并编写该功能在所有平台都会使用到的通用代码
  2. 在 platforms 目录下对应文件夹中创建子类、在子类中编写该功能在该平台的实现逻辑

# 2.2 编译

和编译相关的目录主要是 build 和 config 目录 , build 和 config 目录下的所有文件用于配置 SDK 不同发布环境的编译配置项和 SDK 在不同发布环境不同版本的编译脚本。
