# 1、发布方式

SDK 支持通过 npm 安装来使用和通过 Script 标签引入的方式使用，二者发布方式不一样，这里将两种方式分开说明

# 2、Script 标签引入方式发布

首先编译 SDK ，然后提交最新的 SDK 编译文件，最后在所有使用了 SDK 的项目中重新引入 SDK 。

1、运行 npm run build 命令生成最新版 SDK

2、引入了 SDK 的项目重新引入即可

# 3、NPM 包方式发布

首先编译 SDK，然后发布 NPM 包到 https://www.npmjs.com/package/jw-trigger (opens new window), 最后在所有使用了 SDK 的项目中升级 SDK 或者重新引入 SDK。

1、修改 package.json 文件中的版本号，即修改 version 字段，SDK 版本号按照以下规则设置

版本号命名规则:
1、由主版本号、次版本号、补丁版本号、标签三部分组成
2、主版本号（major）：软件做了不兼容的变更（breaking change 重大变更）；
3、次版本号（minor）：添加功能或者废弃功能，向下兼容；
4、补丁版本号（patch）：bug 修复，向下兼容。
5、标签（label）：
   dev：开发版本
   beta：测试版本
   stable: 稳定版本

2、运行 npm run build 命令生成最新版 SDK

3、运行 npm login 命令登录 NPM 发布账号

4、运行 npm publish 命令发布最新版 NPM 包

5、在引入了 SDK 的项目使用 npm update 命令升级 SDK 或者重新安装 SDK 即可
