# 1、介绍

暂时没有提供 CDN 等公共静态资源服务，通过 Script 标签引入采集探针需要先下载最新的 JavaScript 文件，当前 SDK 各个版本的 JavaScript 文件随项目保存在 Git 仓库的 lib 目录下，

SDK 编译版本保存路径：

http://git.pflm.cn:8880/transfer/trigger/-/tree/master/lib
1

因为没有使用 CDN 等公共静态资源服务，所以后文引入 SDK 使用以下地址进行说明： 在引入实际项目中时请替换成对应的地址：

http://127.0.0.1:28080/dev/1.0.0-dev/trigger.min.js
1

# 2、引入 SDK

将最新的 JavaSvript 文件下载到项目文件夹内以后通过 JavaSvript 异步加载脚本，以免对网站的打开速度造成影响：

<script>
  (function() {
    var scriptElement = document.createElement("script");
    scriptElement.id="jw-trigger"
    scriptElement.src = "http://127.0.0.1:28080/trigger.min.js?type=script&variable=jw_collector&enviroment=TEST&logLevel=DEBUG";
    var existedScript = document.getElementsByTagName("script")[0];
    existedScript.parentNode.insertBefore(scriptElement, existedScript);
  })();
</script>

1

紧跟在 SDK JavaScript 文件 URL 后面的 type=script&variable=jw_collector&enviroment=TEST&logLevel=DEBUG 字符串是 SDK 在安装引入过程中需要使用到的参数，可以通过修改对应参数的值来实现特定的功能 ,目前 SDK 在安装过程中支持配置的参数如下：

参数名称 参数说明 参数值 默认值 必传
type SDK 引入方式 script 无 是
variable SDK 挂载对象 SDK 的全局对象 无 是
enviroment 引入 SDK 所在的业务环境 DEV、TEST、UAT、PROD 无 否
api 采集数据提交 API 地址 API 地址 无 否
logLevel SDK 内部日志等级 ERROR、WARN、INFO、DEBUG ERROR 否

  1. type字段：SDK 加载过程中会尝试获取该参数用来判断 SDK 是否是通过 Script 标签引入的。如果该参数的值为 script 则表示 SDK 是通过 Script 标签引入，SDK 将会按照在浏览器中通过 Script 标签引入 SDK 的方式来安装采集器、采集数据和上报数据。如果该参数的值未设置或者为其他值，则 SDK 会按照 NPM 包引入的方式安装采集器、采集数据、上报数据。

  2. variable字段：该字段用来向 SDK 表明 SDK 安装完成后需要将 SDK 全局对象挂载到 window 下的某个变量。例如：variable=jw_collector 在 SDK 安装完成后将能从 window.jw_collector 中获取到 SDK 暴露出来的自定义采集方法 record()。

  3. enviroment字段：该字段用来设置引入 SDK 的业务环境，DEV、TEST、UAT、PROD 分别表示开发环境、测试环境、UAT 环境、正式环境，设置该值后 SDK 将向对应环境的数据提交接口提交采集数据。enviroment 字段非必传，如果没有传值则将试图获取 api 参数值。enviroment 参数与 api 参数至少需要传一个，否则将会抛出异常。

  4. api字段：当 enviroment 字段没有传值时 SDK 将从该字段获取 SDK 采集数据的上传地址。

  5. logLevel字段：该字段用于设置 SDK 内部的日志等级、当 SDK 输出的日志等级大于或等于该等级时将被输出，否则将不会输出。SDK 日志等级包括 ERROR、WARN、INFO、DEBUG 四个等级，其等级 ERROR > WARN > INFO > DEBUG ,所以设置日志等级为 INFO 级时 ERROR、WARN、INFO 级别的日志将会输出，DEBUG 级别的日志不会输出。

# 3、使用

SDK 在被引入了以后将会自动采集设备信息、上报PV、UV，记录用户访问路径和页面停留时长。除此之外如果需要采集自定义的数据时需要使用 record() 函数。 record() 函数的声明如下：
