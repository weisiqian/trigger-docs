# 1、采集数据结构

数据采集后向后端提交时的数据结构如下所示。

{
  "data": [
    {
      "type": "采集数据类型",
      "data": {
        "key1": "value1",
        "key2": "value2"
        ...
      }
    }
  ],
  "platform": "SDK 运行平台",
  "type": "数据提交方式"
}

提交采集数据实际上是向后端发送一个 JSON 格式的字符串。这个 JSON 格式的字符串是一个对象，对象使用 key、value 的形式表明采集方式和采集到的内容。对象具有层级结构，当前除自定义采集方式外其他采集方式的每个对象都使用了三层结构，第一层用来描述采集方式的相关信息、第二层用来描述数据采集类型的相关信息、第三层用来记录采集到数据的详细内容，每一层都使用 key、value 的形式描述。

# 1.1 对象第一层结构

第一层用来描述采集方式的相关信息、比如当前采集数据是在什么平台采集的、当前采集数据的提交方式是什么，目前第一层使用到了以下几个字段：

  1. type字段：该字段表明发送采集数据的方式，目前有且仅有两个可选值，分别是 “sendBeacon”、“ajax”，值为“sendBeacon”时表明该请求是通过 navigator.sendBeacon() API 发送的，同理值为“ajax”则表示该请求是使用 AJAX API 发送的。

之所以需要使用 “sendBeacon”、“ajax”两种方式提交数据是因为考虑到向下兼容的需求而做出的 API 向下降级，因为 SDK 存在页面关闭后再提交采集数据的需求，如：页面关闭后提交用户在页面的停留时长、页面关闭后提交用户访问路径，目前主要有以下方式可以在页面关闭过程中或关闭后提交数据。

  1. 使用 GET 请求 1*1 大小图片的方式提交数据（最大提交数据大小受 GET 请求的限制、会阻塞页面的关闭流程）
  2. 使用同步 AJAX 的方式提交数据（POST 请求没有大小限制、但会阻塞页面的关闭流程）
  3. 使用 navigator.sendBeacon() API（不会阻塞页面的关闭流程、但低版本浏览器存在兼容性问题）

在比较了各种页面关闭提交数据的方式后发现不存在完美支持关闭页面提交数据的 API，，综合考虑后采用在支持 navigator.sendBeacon() API 的浏览器使用 navigator.sendBeacon() API 提交采集数据而在其他浏览器降级使用同步 AJAX 提交的方式。

  1. platform字段：该字段表明引入 SDK 的平台，目前有且仅有两个可选值，分别是 “NativeJS”、“Vue”，分别表示使用 script 标签引入 SDK 和在 Vue Cli 项目中引入 SDK。

  2. data字段：该字段保存采集到的所有数据，data 字段的值是一个数组，数组中的每一个元素都是 JSON 对象，里面描述采集对象的第二层结构、目前只有 data 字段有第二层结构，data 字段的结构如下所示。

# 1.2 对象第二层结构

第二层用来描述数据采集类型的相关信息、该层是一个数组，数组中的每一个元素都是 JSON 对象，每个对象记录一种采集方式的采集数据，主要包含当前采集数据所属的采集类型，以及采集到的数据内容，目前主要用到以下几个 key：

  1. type字段：该字段表示采集数据的类型，目前有以下几种类型：
类型 名称 说明
PV 页面访问量 用户每进入一次页面算一次访问
UV 用户访问量 同一用户多次进入同一页面只算一次访问
DEVICE_INFO 用户访问设备信息 记录用户浏览器、系统等设备信息
ROUTE_HISTORY 用户访问路径 记录用户从进入页面开始依次访问的所有页面
PAGE_STAY_DURATION 页面停留时长 记录用户在页面停留的时长
CUSTOMIZED 自定义采集数据类型 根据需求自定义需要采集的数据
  1. data字段：该字段表示采集到的数据内容，里面描述采集对象的第三层结构、目前只有该字段有第三层结构，该字段的结构如下所示

# 2、所有采集类型数据的结构

# 2.1 PV

PV 采集类型数据结构如下所示：

{
  "data": [
    {
      "type": "PV",
      "data": {
        "pageUrl": "http://localhost:18080/index.html#12"
      }
    }
  ],
  "platform": "NativeJS",
  "type": "sendBeacon"
}

# 2.3 DEVICE_INFO

DEVICE_INFO 即设备信息，主要统计用户所使用的设备的相关信息，DEVICE_INFO 采集类型数据结构如下所示：

{
  "data": [
    {
      "type": "DEVICE_INFO",
      "data": {
        "UserAgent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/90.0.4430.212 Safari/537.36",
        "DeviceType": "Windows",
        "BrowserName": "Chrome"
      }
    }
  ],
  "platform": "NativeJS",
  "type": "sendBeacon"
}

# 2.4 ROUTE_HISTORY

ROUTE_HISTORY 即用户访问路径，记录用户进入页面后所访问页面的路径，ROUTE_HISTORY 采集类型数据结构如下所示：

{
  "data": [
    {
      "type": "ROUTE_HISTORY",
      "data": {
        "routeList": [
          {
            "routeUrl": "http://192.168.1.40:8083/details/?smsProductInfo=122&smsProductSpecId=163&appkey=2",
            "routeTitle": "商品详情",
            "enterTime": "2021-06-02 16:39:25",
            "leaveTime": "2021-06-02 16:39:33"
          },
          {
            "routeUrl": "http://192.168.1.40:8083/index?appkey=2",
            "routeTitle": "首页",
            "enterTime": "2021-06-02 16:39:33",
            "leaveTime": "2021-06-02 16:42:19"
          }
        ]
      }
    }
  ],
  "platform": "Vue",
  "type": "sendBeacon"
}

用户访问路径采集数据对象第三层为一个数组，数组下标从小到大依次为用户先后访问的每个页面，每个数组元素为用户访问路径中的一个路由节点，每个节点用对象描述了该路由的相关信息，对象的每个 key 的含义和取值如下所示：

key 名称 说明
routeUrl 访问 URL 记录用户所访问页面的 URL
routeTitle 访问页面名称 记录用户所访问页面的名称
enterTime 进入页面时间 记录用户进入该页面的时间
leaveTime 离开页面时间 记录用户离开该页面的时间

# 2.5 PAGE_STAY_DURATION

PAGE_STAY_DURATION 即用户停留时长，记录用户从进入该页面开始到离开页面所经过的时长，PAGE_STAY_DURATION 采集类型数据结构如下所示：

{
  "data": [
    {
      "type": "PAGE_STAY_DURATION",
      "data": {
        "pageUrl": "http://192.168.1.40:8083/index?appkey=2",
        "enterTime": "2021-06-02 16:39:33",
        "closeTime": "2021-06-02 16:42:19"
      }
    }
  ],
  "platform": "Vue",
  "type": "sendBeacon"
}

# 2.6 CUSTOMIZED

CUSTOMIZED 即自定义采集数据类型，用于根据需求自定义采集数据，该采集类型需要使用 record() API 进行采集。CUSTOMIZED 采集类型数据结构如下所示：

{
  "data": [
    {
      "type": "CUSTOMIZED",
      "data": "自定义数据"
    }
  ],
  "platform": "NativeJS",
  "type": "sendBeacon"
}
