# 前言

数据采集 SDK 包已经发布到了 NPM 仓库，可以直接使用 npm install jw-trigger命令进行安装，Vue Cli 项目使用 SDK 只需要安装 npm包，然后引入项目进行配置即可。

# 1、安装

使用npm的方式安装，能更方便进行升级。npm安装命令如下所示，cnpm、yarn等其他包管理工具自行替换命令。

npm install jw-trigger --save
1

# 2、引入

在 Vue 项目中，数据采集 SDK 是作为 Vue 插件的形式存在的，所以 Vue 项目中安装数据采集 SDK 其实就是安装一个 Vue 插件。即首先通过 import Trigger from 'jw-trigger' 引入 SDK，然后使用 Vue.use(Trigger,{router: router}) 安装插件，为了更好的采集用户访问路径等信息，所以需要将 Vue Router 实例作为参数传入插件中，SDK 将会自动调用全局导航守卫采集相应的数据，如果没有传入 Vue Router 实例，SDK 将会在控制台抛出异常。main.js 文件中完整的 SDK 引入和配置代码如下所示：

import Vue from 'vue'
import App from './App.vue'
import router from './router'

// 引入 SDK
import Trigger from 'jw-trigger'
import {
  SdkEnviroment,
  SdkPlatform,
  Collector
} from 'jw-trigger';

// 配置 SDK
Trigger.platform = SdkPlatform.Vue;      // 设置 SDK 引入平台为 VUE
Trigger.enviroment = SdkEnviroment.TEST; // 设置 SDK 运行环境为测试环境
Trigger.logLevel = "DEBUG";              // 设置 SDK 内部日志等级为 DEBUG 级别
Vue.use(Trigger, {                       // 安装 SDK
  router: router
})

Vue.config.productionTip = false

new Vue({
  router,
  render: h => h(App)
}).$mount('#app')

上述代码块中的 SdkEnviroment 是出于 SDK 拓展性的考虑而设置的一个 SDK 环境变量，该变量的值是枚举类型，用来区分不同框架的项目和 SDK 所处的运行环境，后续如需拓展支持 React、Angular、微信小程序等其他框架的话可以通过扩充该变量的枚举值来区分不同的项目运行环境，从而实现特定平台的功能。目前该环境变量只有 SdkEnviroment.Vue 和 SdkEnviroment.NativeJS 两个枚举值，Vue Cli 项目所使用的值为 SdkEnviroment.Vue。

Vue Cli 项目如果不设置 SdkEnviroment.Vue 的话 SDK 将不能正常使用。各参数的用法见SDK 引入参数

# 3、使用

SDK 在被引入了以后将会自动采集设备信息、上报PV、UV，记录用户访问路径和页面停留时长。。除此之外如果需要采集自定义的数据时需要使用 record() 函数。 record() 函数的声明如下：

  /**
   * 记录自定义采集数据
   * @param data 待上传采集数据
   */
  async record(data: string) {

  }
